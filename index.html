      It is suggested that I set the www.customdomain.com as the primary domain in order to benefit from the full advantages of a CDN. I attempted to set the www subdomain as the primary while the status was "Awaiting External DNS." Following the steps, Options (dropdown) → "Set as primary domain" → a confirmation modal appears → "Set as primary domain".

      However, after confirming, the modal updated with a warning: "We're provisioning a certificate for your site, and you cannot change custom domains until that process completes." Subsequent clicks on 'Set as primary domain' within the modal had no effect.

      Interpreting the warning, it seemed that a background process was in progress to facilitate my transaction. Despite waiting for some time, I received no additional information. To investigate further, I opened DevTools, examined the network tab, and discovered a 422 code along with the error/warning message present in the modal.

      Believing that resolving the "Awaiting External DNS" status was necessary, I followed the steps to clear it. Once the DNS records were successfully recognized by Netlify, I could then set the subdomain as my primary domain without any issues. Notably, this occurred on the Chrome browser.

      A suggested fix to this issue would be to disable the 'Set as primary domain' button while the 'Awaiting External DNS' status is present. Additionally, providing a clear indication to the user that external records must be set before attempting to set the www subdomain as the primary would enhance the user experience.
